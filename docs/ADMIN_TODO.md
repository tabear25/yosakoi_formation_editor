## 1. チーム定義（`APP_TEAMS`）を用意する

実際に使うチームの ID / パスワードを JSON 配列で決めます。

```json
[
  {"id":"teamA","password":"ここに強いパスワード","name":"チームA"},
  {"id":"teamB","password":"別の強いパスワード","name":"チームB"}
]
```

ルール（重要）:
- `id` は **半角英数・`-`・`_` のみ、1〜64文字**（KV キー `doc:<id>` に使われるため）。日本語・空白・記号は不可。
- `id` は重複不可。一度決めた `id` を後で変えると、そのチームの保存データ（`doc:<id>`）が参照できなくなるので注意。
- `password` は各チームで十分に強いものを設定。
- `name` は任意（将来の表示用。なくても動作する）。

> Vercel の環境変数には**1行のJSON**として貼り付けます（改行なし）。例:
> `[{"id":"teamA","password":"...","name":"チームA"},{"id":"teamB","password":"...","name":"チームB"}]`

---

## 2. Vercel の環境変数を更新する

Vercel プロジェクト → **Settings → Environment Variables** で以下を設定します。

| 操作 | 変数 | 値 |
| --- | --- | --- |
| **削除** | `APP_LOGIN_ID` | （旧・不要） |
| **削除** | `APP_LOGIN_PASSWORD` | （旧・不要） |
| **追加** | `APP_TEAMS` | 手順1で作った JSON（1行） |
| 確認 | `SESSION_SECRET` | 既存のままでOK。未設定なら下記で生成して追加 |
| 確認 | `KV_REST_API_URL` / `KV_REST_API_TOKEN` | KV（Storage）連携で自動注入済みのはず |

`SESSION_SECRET` を新たに作る場合の生成例:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

> ⚠️ `SESSION_SECRET` を変更すると、既存の全ログインセッションが無効になり全員が再ログインになります（データは消えません）。特に理由がなければ既存値を維持してください。

---

## 3. 既存データ（`doc:main`）の移行（必要な場合のみ）

これまで運用していて `doc:main` に保存済みの表がある場合、その内容は新方式では参照されません。引き継ぎたいときは、KV（Upstash）コンソールで **`doc:main` を `doc:<引き継ぎ先teamId>` にコピー/リネーム** してください。

- Upstash の Redis CLI 例: `RENAME doc:main doc:teamA`
- まだ実運用データが無い／不要なら、この手順はスキップでOK。

---

## 4. メール文面（mailto）を運用に合わせて調整する

ログイン画面の「アカウント発行を依頼する」リンクの宛先・件名・本文は、次のファイルの**先頭の定数**にまとまっています。書き換えたいときはここだけ編集すれば反映されます。

- ファイル: `src/components/LoginGate.tsx`
- 定数:
  - `REQUEST_ACCOUNT_EMAIL` … 宛先メールアドレス
  - `REQUEST_ACCOUNT_SUBJECT` … 件名
  - `REQUEST_ACCOUNT_BODY` … 本文テンプレート（チーム名／ご担当者名／ご連絡先／備考）

> 宛先を別アドレス（例: 専用の受付用アドレス）にしたい場合もここを変更します。変更後は再ビルド／再デプロイが必要です。

## 6. 動作確認チェックリスト

Preview もしくは本番で確認します（ローカルなら `vercel dev` + `.env.local`）。

- [ ] teamA の ID/PW でログインできる
- [ ] teamA で編集 → 自動保存（ヘッダーが「保存済み」）→ リロードで復元される
- [ ] ログアウト後、teamB の ID/PW でログイン → **teamA の表が見えない**（別データになっている）
- [ ] KV に `doc:teamA` と `doc:teamB` が別キーで作られている
- [ ] 間違った ID/PW では「ID またはパスワードが違います」になる
- [ ] ログイン画面の「アカウント発行を依頼する」をクリック → 宛先・件名・本文が埋まったメール下書きが開く
- [ ] 合言葉での共有（読み取り専用ビュー）が従来どおり開ける
